###목표 기능

```
수정날짜, 생성날짜, 이름, 크기 등등으로 정렬하여 표시하는 기능
```
```
스크롤 한 부분의 썸네일 만을 다운로드 하는 기능
 - 페이지 로딩
 - 다만 전체 썸네일 프리캐싱? 기능이 있으면 좋겠음. 버튼 등 이용
```
```
썸네일 크기를 지정해서 원하는 비율로 보여주는 기능
 - 표시 크기에 따라서 썸네일 따로 저장? 고민해봐야 할 부분임
```
```
여러 개의 썸네일을 선택하여 파일 등을 이동하는 기능이 필요함
 - 여러개의 이미지를 웹에서 선택 어떻게 하지?
 - 이동 삭제 등으로 이미지를 처리했을 때 썸네일 등 메타정보도 처리해 줘야함
```
```
동작에 웹소켓을 이용했으면 좋겠음
 - 썸네일을 다운받을 때 각각의 썸네일을 http 패킷으로 주고받으면 효율적이지 못하니. 조금이라도 더 빠르게 처리하기 위해 웹소켓을 이용하는 방법이 있었으면 좋겠음.
 - 다만 웹소켓 특성상 페이지 벗어나면 연결 끊어지는 점에 주의 필요
 - base64 이미지 인코딩을 이용하면 웹소켓을 이용하여 이미지를 쉽게 전송할 수 있다. 아주 좋은방법인것같음
```
```
폴더 생성 등의 부가기능
 - 필수는 아니지만 있으면 좋겠다.
```
```
썸네일을 클릭했을 때 원본 이미지를 표시하는 기능
 - 당연히 필요한 기능이지만 생각해보니까 이미지를 드래그했을 때 다음 이미지나 이전 이미지가 나와야 하는거 아닌가? 어떻게 구현하지?
```
